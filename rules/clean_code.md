# Clean Code & Naming Conventions

1. **Language Strictness:** ALL variables, function names, database columns, and code comments MUST be written entirely in English. **NO Turkish characters (ç, ğ, ı, ö, ş, ü) are allowed anywhere in the codebase.**
2. **Meaningful Names:** Use descriptive, searchable names. (`calculateAverageTime` instead of `calcAvg`).
3. **KISS & DRY:** Keep It Simple, Stupid. Don't Repeat Yourself. Extract reusable logic into helper functions.
4. **Formatting:** Use Prettier standards.
5. **Comments:** Explain the *WHY*, not the *WHAT*. The code should explain what it is doing.
